KMP algorithm

Knuth-Morris-Pratt String Search Algorithm is a Fast String Search algorithm.

Fast

Compared to the naive string search

index 0 1 2 3 4 5
Time A A A A A B
1 A A A B
2 A A A B
3 A A A B

In which the comparisons is performed on the indices [0,1,2,3,1,2,3,4,2,3,4,5]

KMP’s idea :: Compare not the part that is already compared.

Ideally, No need to look back, compare on the indices [0,1,2,3,3,4,4,5]

KMP’s Method :: An offset table D based on pattern

Both versions use a table storing a value D[j] of where to go after spotting mismatch on S[i] and P[j]

(setq S '(:a :b :c :a :b :c :a :b :e))
(setq P '(:a :b :c :a :b :e))
(defun tablev1 (P)
  "return the ver1 table computed from P"
  (let ((j 1) ;; start comparison at index 1
	(k 0) ;; lenght of identical suffix/prefix is 0 originally
	(D (make-vector (length P) 0)))

	(while (< j (length P))
	  (if (equal (nth j P) ;;P[j]
		     (nth k P) ;;P[g(k)] where g(k) = k.
		     ;;g(k) map the length k to index right after that long of string
		     ;;e.g. AB[C]......AB[E]...
		     ;;        k          j
		     ;;so if under a scheme where arrays start with 1, g(k) = k + 1.
		     )
	      ;;if P[j]=P[k], increment j and k, and assign D[j] the corresponding
	      ;;identical suffix/prefix length k.
	      (progn (incf k)
		     (aset D j k)
		     (incf j)
		     (message "inc")
		)
	    ;;    if P[j] # P[k], reset the found length to D[k]
	    ;; (shifting back to a place where a shorter identical suffix/prefix exist, if any, at P[k], for P[0]...P[k-1]), and restart the process
	    (progn (incf j)
	      (setq k (aref D k)))
	    )
	  )
	D
	)
  )
(tablev1 P)
